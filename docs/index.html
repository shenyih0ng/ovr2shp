Basic Usage

./ovr2shp [-t] [-d] [-p] [-o] src

OptionsDescription
-tDisplay HFA Tree structure of input .ovr file
-dDump of annotations (metadata & geometry data)
-pPlot annotation geometries using gnuplot
-srsOutput coordinate reference system. proj4 standard
-o <o_shpfile>Output .shp file path

Supported ERDAS Annotation Objects#

An ERDAS Annotation Layer can contains up to 18 different types of geometric data.

ovr2shp currently only supports conversion of the following geometric data in a ERDAS Annotation Layer:

EANT_TYPESupportedConverted .shp geomtype
EANT_FILLED_POLYGON
EANT_VECTORS
EANT_GRID_CELLS
EANT_UNFILLED_POLYGON
EANT_PATTERN_POLYGON
EANT_FILLED_CIRCLE
EANT_UNFILLED_CIRCLE
EANT_PATTERN_CIRCLE
EANT_ARC
EANT_TEXTYESPOINT
EANT_SYMBOL
EANT_LINE_GRID
EANT_RECTANGLE
EANT_ELLIPSEYESPOLYGON
EANT_POLYGON
EANT_POLYLINEYESPOLYGON
EANT_GROUP
EANT_POINT

Due to the ERDAS IMAGINE being a paid software, it is hard to obtain open and free data to do development. Additionally HFA being a proprietary file format, there no open standard to how different EANT_TYPEare implemented. Thus, data conversions for different geometric types are heavily dependent on the availability of data.
